## Overview

Added Agile daily standup markdown file generation to the exarp-go automation tool's daily action. When `automation(action="daily")` is called, it now automatically generates a structured daily standup markdown file following Agile methodology practices.

---

## Features

### ‚úÖ Implemented

1. **Daily Standup MD File Generation**
   - Automatically creates `daily-standup-YYYY-MM-DD.md` file
   - Located in the same directory as the daily automation report (typically `docs/`)
   - Structured for Cursor to process and read

2. **Agile Standup Template Sections**
   - **What I Did Yesterday**: Lists recently completed tasks (last 24 hours)
   - **What I'm Doing Today**: Shows In Progress tasks and high-priority Todo tasks
   - **Blockers / Impediments**: Identifies blocked tasks and failed automation tasks
   - **Automation Summary**: Includes daily automation statistics

3. **Task Intelligence**
   - Pulls data from Todo2 state file
   - Filters to current project tasks
   - Prioritizes tasks by status and priority
   - Includes task recommendations from agentic-tools (if available)

---

## Usage

### Recommended: exarp-go (Go)

```bash
exarp-go -tool automation -args '{"action":"daily"}'
```

Daily standup output is written to `docs/` when using the Go automation tool.

### Via MCP Tool (Cursor)

```python
# Call automation tool with daily action (uses native Go when exarp-go MCP is configured)
automation(
    action="daily",
    output_path="docs/DAILY_AUTOMATION_REPORT.md"  # Optional
)
```

### Output Files Generated

1. **Daily Automation Report**: `docs/DAILY_AUTOMATION_REPORT.md`
   - Standard automation report with task execution results

2. **Daily Standup**: `docs/daily-standup-YYYY-MM-DD.md`
   - Agile standup format for team communication
   - Example: `docs/daily-standup-2026-01-07.md`

## Agile Methodology Alignment

### Daily Standup Format

Follows the standard Agile daily standup format:

1. **What I did yesterday** (Past)
   - Completed work from previous day
   - Achievements and progress

2. **What I'm doing today** (Present)
   - Current work in progress
   - Planned tasks for today
   - Priority-driven task selection

3. **Blockers / Impediments** (Future)
   - Issues preventing progress
   - Dependencies and blockers
   - Automation failures

### Benefits

- **Consistency**: Standardized format for team communication
- **Automation**: No manual entry required
- **Context**: Rich task information from Todo2
- **Cursor Integration**: Structured markdown for AI processing
